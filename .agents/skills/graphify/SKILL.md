# Graphify Context Skill

This skill ensures the agent understands the architectural relationships within the Kiora Backend before taking action.

## When to use this skill

- Use this when asked to analyze the architecture of Kiora Backend.
- Use this when proposing significant architectural changes, such as creating new microservices.
- Use this when integrating new tools that require understanding how different services communicate.
- Use this when you need context about database connections and HTTP contracts between services.

## How to use it

Before writing any code or proposing architectural changes, you MUST follow these steps:

1. **Read the Knowledge Graph:** Read the contents of `docs/CodeGraph.md`. This file contains the structural mapping of the microservices and their dependencies generated by Graphify.
2. **Generate New Graph (Optional):** If you suspect the graph is severely outdated, run the script `./scripts/generate_graph.sh` to update `docs/CodeGraph.md` before reading it.
3. **Read the Vault Index:** Read `docs/Home.md` to understand the layout of the Obsidian documentation. Follow any relevant links (like `docs/INTER_SERVICE_CONTRACTS.md` or `docs/DEGRADATION_MATRIX.md`) if the task requires specific knowledge about resilience or contracts.
4. **Use Context:** Apply the architectural knowledge gained from these documents to formulate your plan or write code that respects the established microservice boundaries and contracts.
